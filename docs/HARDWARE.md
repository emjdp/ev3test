# 하드웨어 (검증된 배선 + 주의점)

이전 로봇(`/home/emjdp/dev/ev3maze/robot`)에서 실제로 동작이 확인된 배선이다.
**Stage 0 스크립트로 실기에서 한 번 더 확인한 뒤** 다음 단계로 넘어간다.

## 포트 배선

| 역할 | 포트 | ev3dev2 클래스 |
|---|---|---|
| 주행 왼쪽 라지 모터 | `outA` | `LargeMotor` |
| 주행 오른쪽 라지 모터 | `outB` | `LargeMotor` |
| 그립&리프트 미디엄 모터 | `outC` | `MediumMotor` |
| 컬러센서 왼쪽 | `in1` | `ColorSensor` |
| 컬러센서 중앙 | `in2` | `ColorSensor` |
| 컬러센서 오른쪽 | `in3` | `ColorSensor` |
| 초음파 센서 | `in4` | `UltrasonicSensor` |

- **중앙 컬러센서(in2)**: 라인추종(반사광 모드) + 노드 색 판정(컬러 모드) 겸용.
- **좌/우 컬러센서(in1/in3)**: 분기(노드) 형태 감지용. Stage 3부터 사용.

## 센서 모드 메모

- 라인추종은 `ColorSensor.reflected_light_intensity` (0~100, 반사광)을 쓴다.
  흰 바닥 = 큰 값, 검은 선 = 작은 값.
- 노드 색 판정은 `ColorSensor.color` (정수 코드)를 쓴다:
  `0=없음 1=검정 2=파랑 3=초록 4=노랑 5=빨강 6=흰색 7=갈색`
- 반사광 ↔ 컬러 모드 **전환 직후 값이 튄다.** 전환 후 짧게 대기 + 더미 읽기 몇 번 필요.
  (이건 Stage 4에서 다룬다. Stage 1~3에서는 반사광/색을 섞지 않는다.)

## 모터 메모

- 좌우 라지 모터는 마찰·배터리·바퀴 편차로 **같은 명령에도 다르게 돈다.**
  직진 쏠림은 좌우 속도 트림으로, 회전 각도는 회전 시간/속도로 **따로** 잡는다.
- 회전 각도는 코드상 각도와 실제가 다르므로, **좌90·우90·U턴을 각각 따로 보정**한다
  (Stage 2). 자이로 없이 시간/속도 기반으로 맞춘다.

## 안전

- 첫 실행은 **속도를 낮게**(예: 15~20%) 시작한다.
- 브릭 **뒤로가기(BACK) 버튼은 프로그램 입력으로 할당하지 않는다.** ev3dev 기본 종료
  동작으로 남기고, 주행 정지는 네트워크 `stop`/키보드 인터럽트/별도 안전 입력으로 처리한다.
- 모터가 멈춰도 관성으로 밀린다. 정지 후 짧은 settle 대기를 둔다.
